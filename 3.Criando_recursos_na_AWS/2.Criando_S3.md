# Criando S3 

<div align="justify">

O Amazon S3 (Simple Storage Service) é um serviço de armazenamento de objetos. Ele guarda arquivos (objetos) dentro de buckets, que funcionam como pastas na nuvem. 
Cada objeto possui: 

* **Dados** (o conteúdo do arquivo);

* **Metadados** (informações sobre o arquivo, como tipo e data);

* **Chave** (identificador único dentro do bucket);

### :building_construction: Como criar um bucket (passo a passo) 

 

1. Acesse o Console AWS e abra o serviço S3. 

2. Clique em “Criar bucket” (Create bucket). 

3. Defina o nome do bucket (único globalmente, seguindo as regras de nomenclatura). 

4. Escolha a região AWS onde o bucket será criado (por exemplo, us-east-1).  
A região influencia latência, custo e conformidade. 

5. Configurações principais:  
Bloquear acesso público (recomendado por padrão).  
Habilitar versionamento (opcional, mas boa prática).  
Criptografia padrão (AES-256 ou AWS KMS) — importante para segurança. 

6. Clique em “Criar bucket”.  
Agora é possível fazer upload de arquivos (objetos) nesse bucket e controlar permissões.


### :closed_lock_with_key: Segurança e Controle de Acesso 

Por padrão, todos os buckets e objetos são privados. 
O acesso pode ser concedido por: 

* **Políticas de bucket** (JSON baseado em IAM);

* **Políticas do IAM** (baseadas em usuários, grupos ou funções);

* **ACLs (listas de controle de acesso)** — uso legado, hoje desaconselhado;

* **Pontos de acesso S3** — endpoints dedicados com regras próprias;

### :brain: Boas práticas para o dia a dia: 

* Mantenha o bloqueio de acesso público ativado. 

* Use políticas de bucket em vez de ACLs. 

* Utilize criptografia automática para todos os objetos. 

* Ative o versionamento para recuperar versões antigas. 


### :jigsaw:Tipos de buckets  

 
 <span style="color: blue;"><em>*Buckets de Uso Geral*</em></span>

 * Mais comum e padrão. 

* Armazenam qualquer tipo de objeto (imagens, vídeos, documentos etc.). 

* Podem conter quantos objetos quiser e suportam todas as classes de armazenamento (como Standard, IA, Glacier). 

* São privados por padrão, mas podem ser configurados para acesso público. 

* Usados na maioria dos projetos e exemplos da certificação. 

:blue_book: Use quando quiser armazenamento geral e redundância em várias zonas de disponibilidade. 
 
 <span style="color: blue;"><em>*Buckets de Diretório*</em></span>

 * Otimizados para baixa latência e residência de dados (controle de onde os dados ficam). 

* Organizam os dados em uma estrutura hierárquica (como pastas), diferente do formato plano dos buckets de uso geral. 

* Permitem criar até 100 buckets por conta. 

* Podem ser usados em uma única Zona de Disponibilidade (AZ) para obter acesso ultrarrápido (usando a classe S3 Express One Zone). 

:blue_book: Use quando for preciso armazenar dados estruturados para análise e consultas diretas. 

<span style="color: blue;"><em>*Buckets de Vetores*</em></span>

* Projetados para armazenar e pesquisar vetores (usados em IA e machine learning). 

* Permitem realizar buscas por similaridade — útil em sistemas de recomendação, embeddings e IA generativa. 

* Integram-se com serviços como Amazon Bedrock e OpenSearch. 

:blue_book: Use quando precisar trabalhar com embeddings e IA que dependem de vetores. 

### :memo: Versionamento e IDs de Versão 

* Quando ativado, o versionamento guarda múltiplas versões de um mesmo arquivo. 

* Permite recuperar objetos deletados ou substituídos acidentalmente. 

* Cada nova versão recebe um ID único. 

### :earth_africa: Regiões 

 
 Os dados armazenados em uma região AWS nunca saem dela sem replicação explícita. 
A escolha da região deve ser feita considerando latência, custo e requisitos legais. 

 
 ### :gear: Formas de acessar o S3 

 * **Console AWS (interface web)** — ideal para iniciantes. 

* **AWS CLI** — comandos via terminal (aws s3 ls, aws s3 cp, etc.). 

* **SDKs AWS** — acesso programático (Python, Java, .NET, etc.). 

* **API REST S3** — interface HTTP para desenvolvedores avançados. 

### :moneybag: Preço e Free Tier 

* O S3 cobra apenas pelo uso efetivo (armazenamento, requisições e transferência). 

* Free Tier AWS inclui:  
5 GB de armazenamento padrão (S3 Standard)  
20.000 requisições GET  
2.000 requisições PUT
