# Criando Lambda 

<div align="justify">

O AWS Lambda é um serviço que executa código sem precisar gerenciar servidores. 
Ele escala automaticamente, cobra apenas pelo tempo de execução e executa o código sob demanda (event-driven). 

 Preço baseado em uso: você paga apenas pelo número de execuções e tempo em milissegundos. 

 ### :rocket: Casos de Uso

 * **Processamento de fluxo:** trata dados em tempo real (ex: Kinesis Data Streams). 

* **Aplicações Web:** cria back-ends que escalam automaticamente. 

* **Back-ends móveis e IoT:** processa APIs de apps e dispositivos conectados. 

* **Processamento de arquivos:** reage a uploads no S3. 

* **Integração com bancos de dados:** automatiza fluxos de dados quando há mudanças em tabelas. 

* **Tarefas agendadas:** executa rotinas automáticas com EventBridge. 

### :bricks:  Criação de uma Função (exemplo didático) 

1. Acesse o Console AWS → Lambda → Criar função. 

2. Escolha “Criar do zero”. 

3. Defina o nome (ex: myLambdaFunction). 

4. Escolha o runtime (ex: Python 3.13 ou Node.js 22). 

5. O Lambda cria um perfil de execução IAM, com permissões básicas para escrever logs no CloudWatch. 

6. Substitua o código padrão pelo seu código (exemplo: Hello Word!). 

Código de exemplo (Node.js): 

```node.js
export const handler = async (event) => { 

 
  console.log("Hello World!"); 

 
  return "Hello World!"; 
 

}; 
 ```

7. Clique em Implantar para salvar o código. 

8. Crie um evento de teste no formato JSON (pode ser vazio). 

9. Clique em Executar — o resultado será "Hello World!". 

Os logs de execução podem ser visualizados no CloudWatch Logs, permitindo acompanhar a execução e o desempenho da função. 

### :jigsaw: Conceitos Importantes 

| Conceito | Explicação |
|----------|------------|
**Handler** | Ponto de entrada da função (ex: handler(event, context)). 
**Event** | Dados que disparam a função (ex: upload no S3, evento JSON). 
**Context** | Informações de execução (ex: logGroupName, requestId). 
**CloudWatch Logs** | Serviço onde o Lambda grava logs automaticamente. 
**Runtime** | Linguagem usada: Node.js, Python, Java, C#, etc. 
**Perfil de execução (IAM Role)** | Define as permissões da função Lambda. 


### :brain: Boas Práticas 

* Usar IAM Roles com o princípio do menor privilégio. 

* Monitorar com CloudWatch (logs e métricas). 

* Testar eventos localmente antes do deploy. 

* Manter funções pequenas e específicas (uma função = uma responsabilidade). 

* Excluir recursos após testes (função, logs, roles) para evitar custos.
