# Criando instância EC2 

<div align="justify">

O Amazon EC2 (Elastic Compute Cloud) permite criar instâncias, que são servidores virtuais na nuvem AWS, onde o usuário pode escolher o sistema operacional, configurar aplicações e controlar totalmente o ambiente. 

### :gear: Componentes Principais 

| Componente | Função |
|------------|--------|
**AMI (Amazon Machine Image)**|Modelo da máquina virtual — define SO e softwares pré-instalados. 
**Key Pair (Par de chaves)**| Garante acesso seguro à instância: chave pública na AWS, chave privada no computador do usuário. 
**VPC (Virtual Private Cloud)** | Rede virtual isolada que abriga a instância. Cada conta tem uma VPC padrão com sub-redes padrão em cada zona de disponibilidade. 
**Security Group** | Firewall virtual que controla tráfego de entrada e saída. 
**EBS (Elastic Block Store)**| Volume de armazenamento persistente que funciona como o “disco” da instância. 

### :moneybag: Custo e Free Tier 

* A criação de instâncias EC2 pode ser gratuita dentro do Nível Gratuito (Free Tier). 

* O uso gratuito é válido:  

  Por 12 meses para contas criadas antes de 15/07/2025; 

  Por 6 meses para contas criadas depois dessa data. 

* A instância t2.micro ou t3.micro é a elegível para o Free Tier. 

* A cobrança inicia assim que a instância é executada, mesmo que ociosa. 

* Sempre encerre (terminate) após o uso para evitar custos. 

### :book: Tutorial

<span style="color: blue;"><em>*Etapa 1 – Executar uma instância*</em></span>

1. Acesse o Console EC2 → Executar instância (Launch instance).  
Escolha: 

* AMI: Amazon Linux 2 (elegível Free Tier). 

* Tipo de instância: t2.micro. 

* Par de chaves: criar ou selecionar existente (necessário para login). 

* Rede: use a VPC padrão e sub-rede padrão. 

* Security Group: regra SSH (porta 22) — pode deixar 0.0.0.0/0 apenas para testes. 

* Em produção, limite o acesso ao seu IP. 

* Armazenamento: volume raiz EBS de 8 GiB (Free Tier). 

3. Revise e clique em Launch instance. 

4. O estado passa de pending → running. 

<span style="color: blue;"><em>*Etapa 2 – Conectar-se à instância*</em></span>

* Use SSH (para Linux) ou RDP (para Windows). 

* Exemplo de comando SSH:  
ssh -i "chave.pem" ec2-user@<ip-público> 
 

* Se não conseguir se conectar, verifique regras do Security Group e permissões da chave. 


<span style="color: blue;"><em>*Etapa 3 – Encerrar a instância*</em></span>

 
* No console, selecione a instância → Estado → Encerrar instância (Terminate). 

* Após encerrada, ela não pode ser reativada. 

* Para pausar sem excluir, use Stop, que mantém o volume EBS.  
Obs: Encerrar libera custos. Parar apenas pausa, mas pode gerar cobranças pelo EBS. 

### :bell: Próximas Etapas e Boas Práticas 

* Monitore o uso do Free Tier no console AWS Billing. 

* Configure alarmes do CloudWatch para alertar caso o uso exceda o limite gratuito. 

* Adicione volumes EBS adicionais se quiser mais armazenamento. 

* Evite regras abertas (0.0.0.0/0) em produção. 

* Use tags para identificar e organizar recursos (ex: Name=BrunaEC2).
